Kommando: cherry-pick

Wofür nutze ich cherry-pick?

Wenn man beispielsweise einzelne, sinnvolle Commits aus einem Branch der verworfen wird, übernehmen will, eignet sich gut git cherry-pick. Hiermit kann ein einzelner Commit aus einem anderen Branch "herausgepickt" und in den aktuellen Branch übernommen werden.

Wie verwende ich git cherry-pick ?

Wie bereits am Titel zu erkennen ist, nutzen Sie zum herauspicken eines, oder mehrerer Commits das Kommando git cherry-pick. Im folgenden Beispiel erläutere ich das Kommando visuell.

Schritt 1

Hier sehen wir auf der linken Seite commits des branches "side" und auf der rechten Seite commits des master branches. Wir wollen nun commits des branches "side", nämlich C2 und C4 in den master branch übernehmen.

Schritt 2

Übernehmen Sie nun C2 und C4 in den Master Branch: git cherry-pick C2 C4. Nun befinden sich die commits C2 und C4 auch im master branch. Git hat also einen Patch für die Änderungen des Commits extrahiert und erstellt nun einen neuen Commit im aktuellen, dem master branch. Wie man sieht, ist git cherry-pick sehr einfach in der Benutzung.
