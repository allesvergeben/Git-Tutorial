Kommando: mergen

Wofür nutze ich mergen?

Nachdem Sie einzelne Branches erstellt und an ihnen gearbeitet haben, möchten Sie diese nun natürlich zusammmenführen. Die einfachste Methode hierfür ist: git merge

Wie verwende ich git merge ?

Wie bereits am Titel zu erkennen nutzen Sie zum mergen das Kommando git merge im folgenden Beispiel erläutere ich das Kommando visuell.

Schritt 1

Zunächst zeigt master nun auf einen Commit mit zwei Vorgängern. Wenn Sie den beiden Pfeilen immer weiter folgen, kommen Sie an jedem Commit im Repository vorbei. Das heißt master enthält jetzt alles, was im Repository enthalten ist. Sie können erkennen wie sich die Farben der Commits verändert haben? Um die Vorgänge zu verdeutlichen wurden diese Farbig makiert. Jeder Branch hat seine eindeutige Farbe. Jeder Merge Commit bekommt als Farbe eine Mischung aus den Farben seiner Vorgänger. Sie sehen also, dass die Farbe des Branch master in alle Commits gemischt wurde, die von bugFix aber nicht.

Schritt 2

Mergen Sie nun master in bugFix: git checkout bugFix; git merge master Da der bugFix Branch ein Vorgänger von master war, musste git hier kaum etwas tuen; git verschiebt bugFix einfach auf den Commit, auf den auch der master Branch zeigt. Jetzt haben alle Commits dieselbe Farbe, das heißt jeder Branch enthält die Informationen des gesamten Repositorys!
