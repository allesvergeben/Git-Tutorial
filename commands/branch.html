Kommando: branch

Wofür nutze ich branch?

Branches sind Versionszweige, die einen nebenläufigen Entwicklungsprozess abbilden. Sie sind Verweise auf einen bestimmten Commit. Das Anlegen von Branches verbraucht keinerlei Speicherplatz, dies ist einer der Gründe warum immer mit vielen, kleinen Branches gearbeitet werden sollte. Es wird gewährleistet, dass die Arbeit eines Commits und dessen Vorgängern gesichert wird. Auch in der Teamarbeit sind Branches sehr sinnvoll, da jeder an seinem eigenen Branch arbeitet, die dann später zusammengeführt werden können.

Wie verwende ich git branch?

Zum Anlegen eines Branches wird das Kommando git branch verwendet. Beispiel:

Schritt 1

Unser Repository weist vorerst nur zwei Commits auf. Mit dem Kommando git branch issue, sagen wir, dass ein Branch im Commit issue erstellt werden soll. Der Branch issue zeigt nun auf den Commit C1, dies ist im Bild sichtbar.

Schritt 2

Der Branch issue zeigt nun auf den Commit C1, dies ist im Bild zu erkennen.

Wenn ein bestimmter Branch angesprochen werden soll, auf dem vielleicht ein weiterer Commit erzeugt wird, muss mit git checkout "Branchname", dieser Branch angesprochen werden. Ansonsten wird der Befehl immer auf dem Branch ausgeführt, auf dem man sich gerade befindet. Dies wird durch ein Sternchen * visualisiert.
